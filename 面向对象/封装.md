## 封装介绍

封装（encapsulation）就是把抽象出的数据[属性]和对数据的操作[方法]封装在一起，数据被保护在内部，程序的其他部分只有他用过被授权的操作[方法]，才能对数据进行操作。

## 封装的理解和好处

1. 隐藏实现细节 方法(连接数据库)<--调用(传入参数...)
2. 可以对数据进行验证，保证安全合理

## 封装的实现步骤

1. 将属性进行私有化[不能直接修改属性]
2. 提供一个公共的set方法，用于对属性判断并赋值
   
   public void setXxx(类型 参数名){
   
   		//加入数据验证的业务逻辑
   
   		属性 = 参数名；
   
   }
3. 提供一个公共的get方法，用于获取属性的值
   
   public 返回数据类型 getXxx(){ //权限判断
   
   		return xx；
   
   }

***

==alt+ins快捷键可以快速生成get和set函数，也可以快速生成构造器==

==set方法可以写在构造器中，同样能实现数据验证==
